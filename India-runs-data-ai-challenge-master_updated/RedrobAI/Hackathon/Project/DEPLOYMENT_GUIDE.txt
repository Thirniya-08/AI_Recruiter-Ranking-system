🚀 DEPLOYMENT STEPS

1. VERIFY INSTALLATION
   ───────────────────
   
   Run this test:
   └─ cd your_project_directory
   └─ python -c "import fastapi, uvicorn, docx, fitz; print('✓ All dependencies OK')"
   
   Expected output:
   └─ ✓ All dependencies OK


2. START THE SERVER
   ────────────────
   
   Option A (Windows batch file - Easiest):
   └─ Double-click: run_fastapi.bat
   
   Option B (Command line):
   └─ python app_fastapi.py
   
   Option C (Advanced with auto-reload):
   └─ uvicorn app_fastapi:app --host 127.0.0.1 --port 8000 --reload
   
   Expected output:
   └─ INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)


3. VERIFY SERVER IS RUNNING
   ────────────────────────
   
   In browser, visit:
   └─ http://127.0.0.1:8000
   
   Should see: Redrob AI Candidate Ranking Dashboard
   
   Or check API:
   └─ http://127.0.0.1:8000/docs
   
   Should see: Swagger UI with all endpoints


4. TEST A WORKFLOW
   ────────────────
   
   Run the test suite:
   └─ python api_test_examples.py
   
   This will:
   1. Create a sample JD
   2. Upload it
   3. Test all 15 endpoints
   4. Report results
   
   Expected: ✓ All tests completed successfully!


5. UPLOAD YOUR FIRST JD
   ──────────────────
   
   Using cURL:
   └─ curl -X POST http://127.0.0.1:8000/api/jd/upload \
        -F "file=@your_jd.pdf"
   
   Or using Python:
   └─ See api_test_examples.py for complete example


6. CONFIGURE PREFERENCES
   ────────────────────
   
   Apply a strategy:
   └─ curl -X POST http://127.0.0.1:8000/api/preferences/strategy/skill_focused
   
   Or set custom:
   └─ curl -X POST http://127.0.0.1:8000/api/preferences/set \
        -H "Content-Type: application/json" \
        -d '{your_preferences_json}'


7. UPLOAD CANDIDATES
   ──────────────
   
   Using cURL:
   └─ curl -X POST http://127.0.0.1:8000/api/upload \
        -F "files=@candidates.jsonl"
   
   System will:
   1. Parse candidate files
   2. Start background scoring
   3. Return upload confirmation


8. RANK CANDIDATES
   ───────────────
   
   Using cURL:
   └─ curl -X POST http://127.0.0.1:8000/api/rank-with-jd \
        -H "Content-Type: application/json" \
        -d '{"top_n": 100}'
   
   Returns: Top 100 candidates ranked by JD + preferences


9. REVIEW & EXPORT
   ────────────────
   
   Get rankings:
   └─ Browse http://127.0.0.1:8000 in web UI
   
   Or export:
   └─ curl -X POST http://127.0.0.1:8000/api/export \
        -H "Content-Type: application/json" \
        -d '{"top_n": 100}' > submission.csv
   
   Result: submission.csv with top 100 candidates

📊 MONITORING & TROUBLESHOOTING

CHECK SERVER IS RUNNING
────────────────────────
Visit: http://127.0.0.1:8000
Should see: Redrob AI dashboard

If 404: Server not running
  └─ Start with: python app_fastapi.py


CHECK API DOCS
──────────────
Visit: http://127.0.0.1:8000/docs
Should see: Swagger UI with endpoints

If 404: Possible import error
  └─ Check console for error messages


SLOW PERFORMANCE
─────────────────
FastAPI should be 2-3x faster than Flask

If slow:
  1. Check system resources (RAM, CPU)
  2. Reduce candidate batch size in jd_manager.py
  3. Increase BATCH_SIZE in app_fastapi.py


MEMORY ISSUES
──────────────
System uses in-memory storage. For large datasets:
  1. Process in batches
  2. Clear data after export: POST /api/clear
  3. Restart server between large uploads


FILE UPLOAD FAILURES
─────────────────────
Check:
  1. File format supported (PDF, DOCX, TXT, JSON, CSV)
  2. File is readable and not corrupted
  3. File size under limit (600MB default)
  4. Python libraries installed: python-docx, PyMuPDF


SKILL EXTRACTION NOT WORKING
──────────────────────────────
Check:
  1. JD file is properly formatted text
  2. Skills mentioned in JD match keywords in jd_manager.py
  3. Add more keyword variations if needed
