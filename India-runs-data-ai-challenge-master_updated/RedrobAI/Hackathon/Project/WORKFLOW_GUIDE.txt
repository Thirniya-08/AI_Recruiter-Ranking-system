DETAILED SKILL MATCHING EXAMPLE
═══════════════════════════════════════════════════════════════════════════════

JD Requirements:
────────────────
Must-Have (5 skills):
  • Python
  • Machine Learning
  • SQL
  • AWS
  • Docker

Nice-to-Have (3 skills):
  • Kubernetes
  • NLP
  • Data Engineering


Candidate Skills:
──────────────────
  • Python ✓ (matches must-have)
  • Machine Learning ✓ (matches must-have)
  • Java (doesn't match)
  • SQL ✓ (matches must-have)
  • Kubernetes ✓ (matches nice-to-have)
  • NLP ✓ (matches nice-to-have)
  (Missing: AWS, Docker from must-have)


Match Calculation:
──────────────────
Must-Have Match:     3/5 = 60%
Nice-to-Have Match:  2/3 = 67%
Preferred Match:     0/0 = N/A

Overall Match % = (60% × 0.6) + (67% × 0.3) + (0% × 0.1) = 62%


Score Impact with Different Strategies:
────────────────────────────────────────
Skill-Focused (Skills 60%):
  Skills impact = 62% × 0.60 = 37.2% of overall score
  Candidate would rank well if other factors are OK

Experience-Focused (Skills 20%):
  Skills impact = 62% × 0.20 = 12.4% of overall score
  Skills matter less, experience/title matter more
  Candidate might rank lower if experience is average

Elite Strategy (Min Skill Match 80%):
  Candidate FAILS filter (62% < 80%)
  Candidate is excluded entirely
  Would need 4/5 must-have skills to qualify
