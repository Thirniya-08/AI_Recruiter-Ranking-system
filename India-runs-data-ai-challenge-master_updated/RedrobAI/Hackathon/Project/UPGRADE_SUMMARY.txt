🚀 QUICK START WORKFLOW

Step 1: Start Server
────────────────────
  run_fastapi.bat
  OR
  python app_fastapi.py

Step 2: Upload Job Description
───────────────────────────────
  POST /api/jd/upload with PDF/DOCX/TXT/JSON/CSV file
  
  Example:
  curl -X POST http://127.0.0.1:8000/api/jd/upload \
    -F "file=@job_description.pdf"

Step 3: Choose Ranking Strategy
────────────────────────────────
  POST /api/preferences/strategy/{strategy}
  
  Options: balanced, skill_focused, experience_focused,
           title_focused, quick_hire, elite

Step 4: Upload Candidate Files
───────────────────────────────
  POST /api/upload with candidates JSONL files

Step 5: Rank by JD Requirements
────────────────────────────────
  POST /api/rank-with-jd
  
  Returns: Candidates ranked by recruiter preferences!

Step 6: Export Top Candidates
──────────────────────────────
  POST /api/export or view results

❓ COMMON QUESTIONS

Q: Can I change the JD after ranking candidates?
A: Yes! Clear JD with POST /api/jd/clear and upload new one

Q: Can I save/load preferences?
A: Currently session-based. Export preferences as JSON for reuse

Q: Do I need to update my frontend?
A: No! All changes are backend-only. Frontend works as-is

Q: What if candidates are in different formats?
A: System now supports JSONL format for candidates (unchanged)

Q: Can I use multiple JDs?
A: Currently one JD per session. Clear and reload for different JD

Q: How accurate is skill matching?
A: Uses keyword matching + semantic similarity. Manual review recommended
