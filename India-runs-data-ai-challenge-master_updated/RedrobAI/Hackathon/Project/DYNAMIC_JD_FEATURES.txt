================================================================================
                    NEW CAPABILITIES ADDED
================================================================================

Your FastAPI application now supports advanced recruiter-defined ranking:

1. ✅ DYNAMIC JOB DESCRIPTIONS
   • Upload JD in any format: PDF, DOCX, TXT, JSON, CSV
   • Automatic skill extraction (must-have, nice-to-have, preferred)
   • Metadata extraction (job title, company, experience required)
   • Replace old fixed JD with custom recruiter JD

2. ✅ FLEXIBLE SKILL MATCHING
   • Intelligent skill extraction from JD text
   • Semantic matching with candidate skills
   • Category-based matching (must-have 80%, nice-to-have 15%, preferred 5%)
   • Detailed match percentages per candidate

3. ✅ RECRUITER PREFERENCES
   • 6 pre-built ranking strategies:
     - Balanced (default)
     - Skill-Focused (for technical roles)
     - Experience-Focused (for senior roles)
     - Title-Focused (for role transitions)
     - Quick-Hire (urgent hiring)
     - Elite (premium candidates only)
   
   • Custom weight adjustment for any factor
   • Minimum threshold filters (experience, skill match, score)
   • Location preferences
   • Title preferences

4. ✅ SMART FILTERING
   • Filter by minimum experience
   • Filter by skill match percentage
   • Filter by overall score threshold
   • Option to exclude honeypots
   • Location and title-based filtering

5. ✅ ENHANCED RANKING
   • Rank candidates using uploaded JD
   • Apply recruiter preferences to ranking
   • Detailed match breakdown for each candidate
   • Filter candidates by criteria

================================================================================
                        NEW API ENDPOINTS
================================================================================

JOB DESCRIPTION MANAGEMENT
────────────────────────────────────────────────────────────────────────────
POST   /api/jd/upload
       Upload JD file (PDF, DOCX, TXT, JSON, CSV)
       Returns: Job title, company, experience required, extracted skills

GET    /api/jd/summary
       Get current JD summary and all extracted skills
       Returns: Full JD information and skill breakdown

POST   /api/jd/clear
       Clear current JD from memory


RECRUITER PREFERENCES
────────────────────────────────────────────────────────────────────────────
POST   /api/preferences/set
       Set custom preference weights and thresholds
       Accepts: Full RecruiterPreferences model

GET    /api/preferences/get
       Get current preferences

POST   /api/preferences/strategy/{strategy}
       Apply preset strategy: balanced, skill_focused, experience_focused,
       title_focused, quick_hire, elite

GET    /api/preferences/strategies
       Get list of available strategies and recommendations

POST   /api/preferences/reset
       Reset preferences to defaults


ENHANCED RANKING WITH JD
────────────────────────────────────────────────────────────────────────────
POST   /api/rank-with-jd
       Rank candidates using uploaded JD and preferences
       Returns: Top N candidates with JD-specific ranking

POST   /api/candidate-match-score
       Get detailed match score for single candidate vs JD
       Returns: Must-have, nice-to-have, preferred match percentages

POST   /api/candidates/filter-by-jd
       Filter candidates by JD and recruiter criteria
       Returns: Qualified and filtered-out candidates

================================================================================
                       QUICK START EXAMPLES
================================================================================

1. UPLOAD JOB DESCRIPTION
   ────────────────────────
   curl -X POST http://127.0.0.1:8000/api/jd/upload \
     -F "file=@job_description.pdf"

   curl -X POST http://127.0.0.1:8000/api/jd/upload \
     -F "file=@job_description.docx"

   curl -X POST http://127.0.0.1:8000/api/jd/upload \
     -F "file=@job_description.txt"


2. CHECK JD SUMMARY
   ──────────────────
   curl http://127.0.0.1:8000/api/jd/summary
   
   Response includes:
   - Extracted job title
   - Company name
   - Experience required
   - Must-have skills count
   - Nice-to-have skills count
   - Preferred skills count
   - Full skill lists


3. APPLY A STRATEGY
   ──────────────────
   curl -X POST http://127.0.0.1:8000/api/preferences/strategy/skill_focused
   
   Options: balanced, skill_focused, experience_focused, title_focused,
            quick_hire, elite


4. SET CUSTOM PREFERENCES
   ────────────────────────
   curl -X POST http://127.0.0.1:8000/api/preferences/set \
     -H "Content-Type: application/json" \
     -d '{
       "skill_match_weight": 0.5,
       "experience_weight": 0.25,
       "title_match_weight": 0.15,
       "location_weight": 0.05,
       "behavioral_weight": 0.05,
       "minimum_experience": 3,
       "minimum_skill_match": 60
     }'


5. RANK CANDIDATES WITH JD
   ────────────────────────
   curl -X POST http://127.0.0.1:8000/api/rank-with-jd \
     -H "Content-Type: application/json" \
     -d '{"top_n": 100}'
   
   Returns: Ranked candidates with JD-specific matching


6. GET CANDIDATE MATCH SCORE
   ──────────────────────────
   curl -X POST http://127.0.0.1:8000/api/candidate-match-score \
     -H "Content-Type: application/json" \
     -d '{"candidate_id": "CAND_ABC123"}'
   
   Returns: Detailed match breakdown for specific candidate


7. FILTER CANDIDATES BY JD
   ────────────────────────
   curl -X POST http://127.0.0.1:8000/api/candidates/filter-by-jd \
     -H "Content-Type: application/json" \
     -d '{"top_n": 100}'
   
   Returns: Candidates that meet minimum criteria


================================================================================
                       RANKING STRATEGIES
================================================================================

BALANCED (Default)
  Weights: Skills 40%, Experience 25%, Title 15%, Location 10%, Behavior 10%
  Use for: General hiring
  When: You want well-rounded candidates

SKILL-FOCUSED
  Weights: Skills 60%, Experience 15%, Title 10%, Location 10%, Behavior 5%
  Use for: Technical roles, specialized positions
  When: Skills are the most important factor

EXPERIENCE-FOCUSED
  Weights: Skills 20%, Experience 45%, Title 15%, Location 10%, Behavior 10%
  Use for: Senior roles, leadership positions
  When: You prioritize seniority

TITLE-FOCUSED
  Weights: Skills 25%, Experience 20%, Title 35%, Location 10%, Behavior 10%
  Use for: Role transition hiring
  When: Relevant title/background is key

QUICK-HIRE
  Weights: Skills 35%, Experience 30%, Title 20%, Location 10%, Behavior 5%
  Minimum Experience: 1 year
  Minimum Skill Match: 50%
  Use for: Urgent hiring needs
  When: You need candidates fast

ELITE
  Weights: Skills 45%, Experience 35%, Title 10%, Location 5%, Behavior 5%
  Minimum Experience: 5 years
  Minimum Skill Match: 80%
  Minimum Score: 75%
  Use for: Premium candidates only
  When: Only accepting top talent


================================================================================
                    SUPPORTED FILE FORMATS
================================================================================

PDF Files (.pdf)
  • Best for: Professional job descriptions
  • Parsing: Full text extraction from all pages
  • Format: PDFs with text (scanned PDFs may not work well)

DOCX Files (.docx)
  • Best for: Microsoft Word documents
  • Parsing: Text + tables
  • Format: .docx format

Plain Text (.txt)
  • Best for: Simple text job descriptions
  • Parsing: Line-by-line extraction
  • Format: Plain UTF-8 text

JSON Files (.json)
  • Best for: Structured data
  • Parsing: Converts to readable format
  • Format: Valid JSON objects or arrays

CSV Files (.csv)
  • Best for: Spreadsheet data
  • Parsing: Comma-separated values
  • Format: Standard CSV format

NICE-TO-HAVE Skills (Beneficial - weighted 15%)
